Detailed Description

template<class T, int totalbits, int esbits, class FT, PositSpec positspec>
class posit::Posit< T, totalbits, esbits, FT, positspec >

Posit value type Stores the posit in the LSB part of the holding type T as signed integer.

Operations are implemented at 4 levels:

  • Level 1: direct manipulation of bit representation
  • Level 2: decoding of the posit in its componentds (s,k,E,F)
  • Level 3: decoding and manipulation of the posit in full exponent form (s,X,f)
  • Level 4: using the posit backend (softfloat by Unpacked, FPU by BackendFloat, fixed point by BackendFixed)
Template Parameters
Tthe holding type that has to be signed due to complement 2 sign method
totalbitsthe significant bits of posit stored in T right aligned. Due to the 2 complement scheme the MSB bits are extension of the sign
esbitsthe size of the exponent
FTthe backend type that can be: unsigned integer for softposit, float/double for emulation, generic Backend, e.g. fixed point
positspecspecifies the number capabilities
Examples
04_eigen.cpp, and 05_fft.cpp.

◆ twice()

template<class T , int totalbits, int esbits, class FT , PositSpec positspec>
auto Posit::twice ( ) const

returns half (Level 1)

Computes twice as Level 1 for es=0, Level 2 for es != 0.

Level 1 for zero esponentbits

+- 2^(R) 1.xyz == +- 2^(exp) 1.xyz 1.xyz * 2 => 1.0xyz ==> just R++ meaning shift R left up to overflow

Level 2 otherwise

+- 2^(R expmax + E) 1.xyz == +- 2^(exp) 1.xyz 1.xyz * 2 => 1.0xyz ==> just exp++ exp++ ==> E++ if E not maximal otherwise R++ and exp zero
